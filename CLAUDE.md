# Brain Games App - Claude作業ガイド

## プロジェクト概要

脳トレーニングゲーム集アプリ。ゲスト・認証ユーザーどちらでもプレイ可能。
認証ユーザーはスコアが保存され、履歴・プロフィール管理ができる。

詳細な仕様は [SPEC.md](./SPEC.md) を参照。

---

## 技術スタック

| 分類 | 技術 |
|------|------|
| フロントエンド | Next.js 15 (App Router) + TypeScript + Tailwind CSS v4 |
| バックエンド | Supabase (Database + Auth) |
| 認証 | Google OAuth 2.0（Supabase経由） |
| デプロイ | Vercel |

---

## ディレクトリ構成

```
src/
├── app/           # ページ（App Router）
│   ├── games/     # 各ゲームページ
│   ├── history/   # スコア履歴
│   ├── profile/   # プロフィール管理
│   └── context/   # GameContext（グローバル状態）
├── components/
│   ├── auth/      # 認証関連コンポーネント
│   ├── layout/    # ヘッダー・レイアウト
│   └── ui/        # 汎用UIコンポーネント
├── hooks/         # ゲームごとのカスタムフック
├── lib/           # Supabaseクライアント
├── types/         # TypeScript型定義
└── utils/         # ユーティリティ関数
```

## 開発ルール

### 作業前
- **実装前に変更計画を日本語で説明**してから作業を開始する

### コード品質
- TypeScriptの **`any` 型は禁止**。適切な型を定義する
- コミット前に `npm run lint` を実行してエラーがないことを確認する

### ファイル操作
- **ファイルやディレクトリを削除する前に必ず確認**を取る

---

## 仕様変更プロセス

ゲームのルール・動作・UI・データ構造など、SPEC.md に記載された内容を変更する場合は、規模に関わらず以下の手順を踏む。

### 提案者
ユーザー・Claude どちらも提案できる。

### 手順

1. **提案**：変更内容・理由・影響範囲を日本語で説明する
2. **承認**：ユーザーが承認する
3. **実装**：コードを変更する
4. **SPEC.md 更新**：変更内容を反映してコミットする

### 原則
- 承認前に実装を開始しない
- 実装後は必ず SPEC.md を最新の状態に保つ
